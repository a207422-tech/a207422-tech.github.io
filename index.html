SDG 12: Responsible Consumption and Production

Eco Brew Tracker empowers users to minimize their ecological footprint by monitoring precise brewing parameters, reducing domestic coffee waste, and syncing shared sustainability metrics remotely to the community cloud.

Laboratory Works

Lab 1: UI

Building the initial static user interface for the application using Jetpack Compose.

Lab 1 focused on building the initial static user interface for the Eco Brew Tracker application using Jetpack Compose. It provided a solid foundation in declarative UI design, allowing me to master component placement and layout structures before introducing dynamic states. This phase successfully established the clean visual architecture necessary for all subsequent development stages of the application.

Lab 2: Interaction

Introducing interactivity and dynamic state management to capture user inputs in real time.

Lab 2 introduced interactivity and dynamic state management to Eco Brew Tracker by capturing and tracking user inputs in real time. I learned to utilize state tools within Jetpack Compose to manage mutable data, such as tracking precise coffee brewing parameters and ground dosages. This phase successfully transformed a static visual layout into a responsive interface, laying the essential groundwork for shared application workflows.

Lab 3: Material Design

Upgrading the UI by applying Material Design foundations for visual consistency.

Lab 3 focused on upgrading the user interface of Eco Brew Tracker by applying Material Design foundations to ensure strict visual consistency across the application. I learned to implement customized color themes, typography, and standard Material components to enhance the overall user experience and accessibility. This aesthetic refinement successfully bridged the gap between basic programmatic functionality and a polished, professional mobile interface.

Lab 4: Navigation

Implementing complex multi-screen workflows and navigation patterns.

Lab 4 focused on implementing complex multi-screen workflows and navigation patterns within the Eco Brew Tracker app using Navigation Compose. I learned how to structure explicit navigation routes and leverage a shared ViewModel to pass and update UI state seamlessly across different screens. This phase successfully transformed separate design components into a unified, fluid application with smooth transitions and robust state handling.

Lab 5: Room

Introducing local data persistence by integrating the Room Database.

Lab 5 introduced local data persistence into Eco Brew Tracker by integrating the Room Database to save user coffee data permanently on the device. I learned how to configure data access objects (DAOs), entities, and database instances to ensure that the application remains fully functional and accessible offline. This implementation successfully transitioned the app from temporary ViewModel state into a structured, robust local data storage solution.

Major Projects

Eco Brew Tracker (Phase 1)

Problem Statement: Coffee enthusiasts often struggle to maintain consistent brewing parameters, leading to suboptimal extractions and unexpected domestic coffee waste. Eco Brew Tracker solves this by providing a structured logging interface to monitor brewing metrics, encouraging mindful coffee consumption habits.

Project 1 successfully integrated multi-screen navigation with centralized state management using a single shared ViewModel. I mastered the principles of state hoisting by passing transient data across five functional screens and dynamically updating the list UI when a new log was added. This phase reinforced the importance of standardizing design flows and state isolation before transitioning into backend data structures.

Eco Brew Tracker (Phase 2)

Problem Statement: While individual coffee brewing parameters can be tracked locally, a lack of shared community data makes it difficult to coordinate wide-scale waste reduction efforts. Eco Brew Tracker addresses this by utilizing remote cloud synchronization and hardware-aware triggers to encourage collaborative, sustainable coffee consumption.

Project 2 advanced my software engineering skills by requiring the concurrent implementation of local database storage via Room and cloud synchronization via Firebase Firestore. I mastered handling hardware lifecycles by capturing physical movement through the phone's accelerometer and gained extensive experience parsing real-time JSON payloads using Retrofit REST APIs. Ultimately, I learned how to build a scalable architecture that bridges physical sensor inputs with multi-layered data persistence to provide a fluid user experience.
